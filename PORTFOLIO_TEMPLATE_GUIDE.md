# 포트폴리오 디자인 템플릿 가이드 (공유용 — 개인정보 없음)

> 이 파일은 외부에 공유해도 되는 버전입니다. 본인 경력 수치·회사명·연락처 등 민감정보는 전부 제외했습니다.
> `CLOUD.md`(원본 인수인계 메모, 민감정보 포함)는 공유하지 마세요 — 이 파일만 넘기면 됩니다.

---

## 1. 이 가이드 사용법

다른 사람이 본인 텍스트·경력·이미지로 같은 디자인의 포트폴리오를 만들고 싶을 때, Claude Code에게 이 파일을 통째로 주고 아래처럼 요청하면 됩니다:

```
이 가이드(PORTFOLIO_TEMPLATE_GUIDE.md)와 같은 구조·디자인으로 단일 HTML 포트폴리오를 만들어줘.
내 정보는 다음과 같아: [이름/직무/경력 요약/사진 등 본인 자료 첨부]
```

Claude Code는 "디자인 시스템"(2번 CSS 토큰)과 "페이지 구조"(3번 섹션 순서)는 그대로 유지하면서, 본인이 준 텍스트·이미지로 내용만 새로 채워서 결과물을 만들어 줍니다 — CSS 변수와 섹션 순서가 명시돼 있어서 디자인이 흐트러지지 않고 그대로 나옵니다.

---

## 2. 디자인 시스템 (CSS 토큰 — 그대로 복사해서 써도 됨)

색상·여백·모션은 하드코딩하지 않고 아래 CSS 변수(토큰)만 사용합니다. 다른 사람이 색을 바꾸고 싶으면 이 블록의 값만 수정하면 전체 디자인이 일괄 변경됩니다.

```css
:root {
  --bg:     #07101A;
  --bg2:    #0C1925;
  --bg3:    #0A1620;
  --card:   #0F2035;
  --card2:  #132740;
  --a:      #4A9EBA;
  --a2:     #6BBDD6;
  --text:   #E8EDF2;
  --sub:    #B0CADA;
  --muted:  #8BABBF;
  --dim:    #567A92;
  --line:   #1C3550;
  --pad:    1.5rem;                 /* 카드 안쪽 패딩/그리드 간격 */
  --ease:   cubic-bezier(0.25, 0.46, 0.45, 0.94);  /* 인터랙션 곡선 */
  --shadow-hover: 0 10px 30px rgba(0,0,0,.28), 0 0 0 1px rgba(74,158,186,.12);
}
```

> 폰트: `Noto Sans KR` + `Inter` (Google Fonts CDN, 오프라인 시 시스템 폰트로 자동 대체)
> 폰트 크기 **0.75rem 미만 금지** — 가독성 하한선.

## 3. 페이지 구조 (섹션 순서)

```
NAV
→ HERO (핵심 지표 내장, 마우스 호버 시 카운트업 애니메이션)
→ 01 ABOUT (역량 핵심 필러 3~4개)
→ 02 CASE STUDIES (사례 2~3건, 각 사례를 "배경(Problem) → 실행(Action) → 결과(Result)" 구조로 작성)
→ 03 CAREER (좌측 레일 + 카드형 타임라인)
→ 04 PROJECTS/CAPABILITY (대표 프로젝트 3~4개, "문제 → 해결" 구조)
→ 05 GALLERY (작업물 썸네일 카드 + 라이트박스 — 4번 참고)
→ 06 CONTACT (이름·이메일·전화·위치)
→ LIGHTBOX (전체화면 이미지 뷰어, 갤러리와 연동)
```

> 섹션은 자유롭게 추가/삭제 가능하지만, **번호(sec-num)는 항상 연속**되게 재정렬할 것.

## 4. 갤러리 패턴 (작업물 카드 + 라이트박스)

- 카드 1개당 썸네일 이미지 2~4장(2×2 그리드 권장), 클릭하면 라이트박스가 폴더(프로젝트) 전체 이미지를 보여주는 구조.
- 라이트박스 데이터는 `const galleries = {...}` 형태의 단일 JS 객체로 관리 — 갤러리 ID(`g1`, `g2`...)별로 이미지 배열을 보관.
- **이미지 파일명은 전부 영문+숫자(ASCII 안전 경로)로 통일**: `gal-01.jpg`, `gal-02.jpg`... 한글·공백·괄호가 든 원본 파일명은 압축/메일/웹호스팅 전달 과정에서 깨질 수 있어서, 참조용 사본은 항상 ASCII 이름으로 별도 폴더에 복사해 쓴다. 원본은 그대로 보존.
- 썸네일에 `loading="lazy"` 속성은 **쓰지 말 것** — 헤드리스 브라우저나 뷰포트 밖 콘텐츠가 렌더링되지 않는 누락 이슈가 있다.

---

## 5. 기술 제약 (반드시 준수)

- **단일 HTML 파일**로 작성 — 외부 CSS/JS 프레임워크(부트스트랩, 리액트 등) 추가하지 않는다. HTML+CSS+JS를 한 파일에 통합.
- 색상/여백은 위 2번 CSS 토큰만 사용 (인라인 스타일, 하드코딩 색상값 지양).
- 인터랙션: `IntersectionObserver`로 스크롤 reveal 애니메이션, `data-count`/`data-suffix` 속성으로 숫자 카운트업, 라이트박스는 `galleries`/`curGal`/`curIdx` 변수로 상태 동기화.
- 반응형 브레이크포인트: `1160px / 1080px / 900px / 660px` 기준으로 레이아웃 전환.

## 6. 단일 파일 배포본(이력서 업로드용) 만드는 법

이력서 사이트 업로드나 메일 첨부용으로, 모든 이미지를 Base64로 내장한 "폴더 없는 단일 파일" 버전을 따로 만들 수 있습니다.

- 생성 방식: 헤드리스 Chrome으로 이미지를 canvas 리사이즈(최대 가로 1280px) → JPEG 압축(품질 0.78) → Base64 인코딩해서 HTML에 직접 삽입.
- 같은 이미지가 썸네일·라이트박스에 중복 등장하면 배열(`__G[]`)에 1번만 저장하고 양쪽에서 재사용 — 파일 용량을 절반 가까이 줄일 수 있다.
- 원본(`index.html`, 폴더 참조형)은 개발·수정용으로 따로 유지하고, **수정은 항상 원본에서 한 뒤 단일 파일 버전을 재생성**한다 (역순 작업 금지).
